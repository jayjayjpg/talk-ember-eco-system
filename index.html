    What is Ember-CLI?

    the Ember.js command line utility for developing your Ember.js projects

    npm install -g ember-cli
npm install -g bower
// Ready to create a new ember project
ember new fancy-app
    still work in progress in 2014, but official build tool since Ember v1.11.0 released synchronously with Ember.js + Ember-Data ~ 6 weeks release cycle will be around roughly every six weeks to make constant deprecation of old features and easy migrations of your projects possible - projects won't break until you ignored the deprecation warnings for a dozen of minor releases this way. More about safe upgrading and helpful tools for that later If you now want to get started with ember you can get started as follows - install ember cli globally if you are a node package management purist and are actually not using it already you will also have to install bower globally and finally you are ready to create a new ember app with ember new command

    Project Structure Setup with 1 command

    ember new fancy-app

    first app setup with basic project structure is created

    If you are familiar with Ruby on Rails, the project structure of a conventional Ember app will be familiar to you: Routes, Components, Models, Adapters are all tucked away in their specific directories adhering to a strict naming convention Ember knows which subclass powers a component based on its filename. For example, if you have a component called blog-post, you would create a file at app/components/blog-post.js. If your component was called audio-player-controls, the file name would be at app/components/audio-player-controls.js.

    36% of participants of the Ember community survey in March this year said that they wrote at least one addon Strength of the conventional Ember approach for addon developers: Without these conventions the community would have a much harder time to explain to reusers of their dependencies how to configure those, makes it also easier to avoid clashes between different plugins.     Server-side rendering of your Ember app in Node.js

    'Native App'-like interactivity and fast responsiveness on initial page load

    Keep it simple: write code once, use it directly with fastboot

    Benefits to Server-side rendering that content is available to crawlers, cURL, JS disabled, fast response on first load Benefits to JavaScript apps regarding interactivity, offline modus, no page relaods FastBoot works by running your Ember application in Node.js. When a user visits your site, the initial HTML is rendered and served to the user. The very first thing they see is your content. app still available even if JS is turned-off awesome for server-side rendering FastBoot will allow you to deliver the HTML and CSS for a page in your Ember app right away, then allow the JavaScript to take over once it has finished loading. Your Ember app will behave no differently than server-rendered apps when it comes to search engines, mobile users, cURL, or users with JavaScript disabled. write the same code as for your usual ember app on the browser cURL the page
