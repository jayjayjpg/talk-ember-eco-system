What is Ember-CLI?

the Ember.js command line utility for developing your Ember.js projects

you@terminal:~/$ npm install -g ember-cli
you@terminal:~/$ npm install -g bower
// Ready to create a new ember project
you@terminal:~/$ ember new fancy-app
still work in progress in 2014, but official build tool since Ember v1.11.0 released synchronously with Ember.js + Ember-Data ~ 6 weeks release cycle will be around roughly every six weeks to make constant deprecation of old features and easy migrations of your projects possible - projects won't break until you ignored the deprecation warnings for a dozen of minor releases this way. More about safe upgrading and helpful tools for that later If you now want to get started with ember you can get started as follows - install ember cli globally if you are a node package management purist and are actually not using it already you will also have to install bower globally and finally you are ready to create a new ember app with ember new command

Ember CLI and its asset pipeline

  • Raw assets available under one specific url: /assets
  • Minification and concatenation of project and third-party CSS and JS
  • Preprocessors are only one ember install command away
ember install ember-cli-sass
That's it! You're now ready to use your .scss files in your app's directory for styles
Minification is optional for external code sources. preprocessors can be - Sass, CoffeeScript and more More on the preprocessors when we cover Addons

Ember goes Server Side made possible by Fastboot

Server-side rendering of your Ember app in Node.js

write the same code for your Ember + Fastboot app in one go with addon support: ember-network

FastBoot works by running your Ember application in Node.js. When a user visits your site, the initial HTML is rendered and served to the user. The very first thing they see is your content. app still available even if JS is turned-off awesome for server-side rendering write the same code as for your usual ember app on the browser finally be happy with SEO for indexable apps - no awkward google js execution waiting anymore

Community Efforts Become More Effective With Strong Conventions

Ember.js shows how solid conventions can help anyone developing build tools, plugins and addons make their efforts easily accessible and reusable

Without these conventions the community would have a much harder time to explain to reusers of their dependencies how to configure those, makes it also easier to avoid clashes between different plugins. Seldom people really enjoy the pleasures of writing extensie documentation to the fullest, so why not spare them the effort by already existing conventions
